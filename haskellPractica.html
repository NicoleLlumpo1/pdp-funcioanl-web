Teoría rápida

Los conceptos esenciales del paradigma funcional para el parcial.

Composición

Operador (.)

-- (f . g) x = f (g x)
dobleIncremento = (*2) . (+1)
-- dobleIncremento 3 → 8

nombreFalopa = (=='i') . last
-- sin auxiliares, solo composición

Aplicación parcial

sumar :: Int -> Int -> Int
sumar x y = x + y

sumar5 = sumar 5
-- sumar5 3 → 8

-- secciones de operadores:
triple  = (*3)
mayor10 = (>10)
Recursión

Estructura básica

longitud :: [a] -> Int
longitud []     = 0
-- ↑ caso base
longitud (_:xs) = 1 + longitud xs
-- ↑ caso recursivo

Mejora sustentable (parcial)

mejora [] _ = True
mejora (x:xs) prev
  | x < prev       = False
  | x > prev + 3   = False
  | otherwise      = mejora xs x
Orden superior

map / filter / fold

map (+1) [1,2,3]
-- [2,3,4]
filter even [1..10]
-- [2,4,6,8,10]
foldr (+) 0 [1,2,3]
-- 6

any / all / take

any (>10) [1,5,15]  -- True
all even [2,4,6]    -- True
take 3 [1..]        -- [1,2,3]

-- listas infinitas: lazy!
take 3 (filter even [1..])
-- termina ✓
Tipos de datos

data + record syntax

data Animal = Animal {
  nombre  :: String,
  tipo    :: String,
  peso    :: Double,
  edad    :: Int,
  enfermo :: Bool
} deriving (Show)

-- modificar un campo (inmutabilidad):
engorde :: Double -> Animal -> Animal
engorde kilos a = a { peso = peso a + min kilos 5 }

-- proceso = lista de actividades:
type Proceso = [Animal -> Animal]

aplicar :: Proceso -> Animal -> Animal
aplicar ps a = foldl (flip ($)) a ps

Tips para el parcial

  • Cuando dice "sin auxiliares ni recursión" → solo (.) y aplicación parcial.
  • nombreFalopa = (=='i') . last — una línea, cero recursión.
  • Modelar un proceso como [Animal → Animal] y aplicarlo con foldl (flip ($)).
  • take N (filter f listaInfinita) → computable (lazy). length (filter f infinita) → NO termina.
  • Para "la pasó mal": any (>30) . map diasRecuperacion . visitas
  • Recursión: siempre primero el caso base con lista vacía [].
